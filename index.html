Highlight #1 A SHORT HISTORY OF GIT

GitHub is the storage system being used in this course. I honestly did not know much about Git prior to coming into this class so this section was really intriguing and informative. From the reading I learned what Git exactly is and its origin. Git "was born" in 2005 and since then it has gotten faster and more efficient. Git, however, was created from the remains of an old system called BitKeeper. The Linux kernel was the jumping-off point for Git and was around from 1991-2002. The relationship that existed between those that cultivated the Linux kernel and the team that cultivated BitKeeper deteriorated and thus, BitKeeper being free did not last much longer. It was at that time Git was starting to develop and was finished in 2005. Since then, people like us have been using Git to aid them in web development.

Highlight #3 WHAT IS VCS?

The title of this assignment is version control. What exactly does version control mean? This video is a great resource for learning version control 101. It's nice and short but also hits all the major need-to-know points of version control. In short, version control is keeping track of changes made throughout cultivating a file so if there are errors or the developer needs to go back and look into the history of the file then they can do so with ease. This is what we use Git for. As previously mentioned in the blurb about the history of Git, Git has become so fast and efficient for developers and is an excellent tool to use for version control!
